# x402 Payment — How It Actually Works

This document explains the complete payment flow from first principles:
what the blockchain is, what a wallet is, what USDC is, and exactly
what happens on-chain when OpenClaw pays for API access.

---

## Part 1: The Building Blocks

### What is Base?

Base is a blockchain — a public database that no single person controls.
Think of it as a shared ledger that thousands of computers around the world
all hold a copy of and agree on.

```
┌─────────────────────────────────────────────────────────────┐
│                      BASE BLOCKCHAIN                        │
│                                                             │
│  A shared, append-only ledger. Once something is written,  │
│  it cannot be changed or deleted.                           │
│                                                             │
│  Every entry (transaction) is:                             │
│    - Publicly visible to anyone                            │
│    - Permanently stored                                     │
│    - Cryptographically verified                            │
│    - Settled in ~2 seconds                                 │
│    - Costs ~$0.001 in gas fees                             │
│                                                             │
│  Base is built on top of Ethereum — it inherits            │
│  Ethereum's security but is 100x cheaper and faster.       │
└─────────────────────────────────────────────────────────────┘

Chain ID:  8453        (mainnet — real money)
           84532       (Base Sepolia — testnet, fake money for development)
```

### What is a Wallet?

A wallet is not an app. It is a pair of cryptographic keys.

```
┌─────────────────────────────────────────────────────────────┐
│                         A WALLET                            │
│                                                             │
│  PRIVATE KEY                                                │
│  ────────────                                               │
│  A random 32-byte number. Looks like:                      │
│  0x4c0883a69102937d6231471b5dbb6e538ebe...                  │
│                                                             │
│  This is the password. Whoever holds this can do           │
│  anything with the wallet. Never share it. Never log it.   │
│                                                             │
│         │                                                   │
│         │ (one-way mathematical derivation)                 │
│         ▼                                                   │
│                                                             │
│  ADDRESS (public)                                           │
│  ────────────────                                           │
│  0x71C7656EC7ab88b098defB751B7401B5f6d8976F               │
│                                                             │
│  This is the "account number". Safe to share publicly.     │
│  It's where people send money to you.                       │
│                                                             │
│  The blockchain stores balances against addresses,          │
│  not private keys. The private key just proves you          │
│  own the address when you want to spend.                    │
└─────────────────────────────────────────────────────────────┘
```

**There are two wallets in our system:**

| Wallet | Owner | Role |
|---|---|---|
| **Platform wallet** | Us (platform) | Receives USDC from buyers. Sends USDC back for refunds. |
| **OpenClaw wallet** | OpenClaw agent | Holds USDC. Pays for API access. |

**How to create the platform wallet (one-time setup):**

```typescript
// Run this once. Save the output. Never run again.
import { generatePrivateKey, privateKeyToAccount } from 'viem/accounts';

const privateKey = generatePrivateKey();
// → 0x4c0883a69102937d6231471b5dbb...
// SAVE THIS in your secret manager / .env file as PLATFORM_WALLET_PRIVATE_KEY

const account = privateKeyToAccount(privateKey);
console.log(account.address);
// → 0x71C7656EC7ab88b098defB751B7401B5f6d8976F
// THIS is your platform wallet address. Put it in the Agent Card.
```

```
⚠️  The private key IS the wallet.
    If you lose it → you lose access to all funds in it.
    If someone else gets it → they can drain all funds.
    Store it in a secrets manager (AWS Secrets Manager, Doppler, etc.)
    Never put it in source code. Never log it.
```

### What is USDC?

USDC is a stablecoin: 1 USDC = $1 USD, always. It is issued by Circle (the company).

On Base, USDC exists as a **smart contract** — a program that lives on the blockchain
and keeps track of who owns how much USDC.

```
┌─────────────────────────────────────────────────────────────────┐
│              USDC CONTRACT ON BASE (simplified)                  │
│                                                                  │
│  Address: 0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913           │
│                                                                  │
│  Internal ledger (balances mapping):                            │
│  ┌──────────────────────────────────────────┐                   │
│  │ 0xOpenClaw...    →   5,000,000  (= $5)   │                   │
│  │ 0xPlatform...   →   1,200,000  (= $1.20) │                   │
│  │ 0xSomeone...    →  10,000,000  (= $10)   │                   │
│  │ ...                                      │                   │
│  └──────────────────────────────────────────┘                   │
│                                                                  │
│  Key functions:                                                  │
│  transfer(to, amount) → moves USDC from caller to 'to'          │
│  balanceOf(address) → returns balance for that address          │
│                                                                  │
│  USDC uses 6 decimal places:                                    │
│    1,000,000  on-chain units  =  $1.00 USDC                     │
│    5,000,000  on-chain units  =  $5.00 USDC                     │
│      500,000  on-chain units  =  $0.50 USDC                     │
│                                                                  │
│  We store prices in these same units (micro-USDC).              │
│  $1.00 tier → price_usdc_micro = 1,000,000                     │
└─────────────────────────────────────────────────────────────────┘
```

## Part 2: What Happens On-Chain (Step by Step)

When OpenClaw sends USDC to the platform, here is exactly what happens:

### A single USDC transfer, dissected

```
OpenClaw wants to send 1.00 USDC to Platform

Step 1: OpenClaw constructs a transaction
┌─────────────────────────────────────────────────────────────────┐
│  TRANSACTION                                                     │
│                                                                  │
│  from:     0xOpenClawWallet                                      │
│  to:       0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913          │
│            ↑ This is the USDC CONTRACT address, not Platform!   │
│                                                                  │
│  data:     transfer(0xPlatformWallet, 1000000)                   │
│            ↑ Encoded function call: "move 1.00 USDC to Platform"│
│                                                                  │
│  value:    0 ETH  (no ETH is moving, only USDC)                 │
│  gas:      ~65,000 gas units                                     │
│  gasPrice: ~0.001 Gwei on Base  (costs ~$0.001 total)           │
│                                                                  │
│  signature: 0xabc123...  (signed with OpenClaw's private key)   │
└─────────────────────────────────────────────────────────────────┘

Step 2: OpenClaw broadcasts this transaction to Base network
        (sends it to a Base node, which shares it with all other nodes)

Step 3: A Base validator includes it in the next block (~2 seconds)

Step 4: The USDC contract executes transfer(0xPlatform, 1000000)
        ┌──────────────────────────────────────────┐
        │  USDC contract ledger BEFORE:            │
        │  0xOpenClaw...  →  5,000,000             │
        │  0xPlatform...  →  1,200,000             │
        └──────────────────────────────────────────┘
                          ↓
        ┌──────────────────────────────────────────┐
        │  USDC contract ledger AFTER:             │
        │  0xOpenClaw...  →  4,000,000  (-1 USDC)  │
        │  0xPlatform...  →  2,200,000  (+1 USDC)  │
        └──────────────────────────────────────────┘

Step 5: The contract emits a Transfer EVENT in the transaction logs
        ┌─────────────────────────────────────────────────────────┐
        │  TRANSFER EVENT (stored in tx logs forever)             │
        │                                                          │
        │  contract: 0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913  │
        │  event:    Transfer                                      │
        │  from:     0xOpenClawWallet                              │
        │  to:       0xPlatformWallet                              │
        │  value:    1000000  (= $1.00 USDC)                      │
        └─────────────────────────────────────────────────────────┘

Step 6: Transaction is now on-chain. OpenClaw receives a tx_hash:
        0x7f3a9b2c4e1d8f5a6b7c8d9e0f1a2b3c4d5e6f7a8b9c0d1e2f3a4b5c6d7
        This hash uniquely identifies this specific transaction forever.
```

## Part 4: What the Platform Reads From the Blockchain

When the platform calls `getTransactionReceipt(tx_hash)`, this is what it gets back from Base and how it reads it:

```
RAW TRANSACTION RECEIPT (what viem returns)
──────────────────────────────────────────

receipt.status          → "success"  or  "reverted"
                           ↑ Must be "success". If reverted, the transfer never happened.

receipt.from            → "0xOpenClawWalletAddress"
                           ↑ Who broadcast the transaction.

receipt.logs            → Array of every event emitted during this tx
   [
     {
       address: "0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913",
                ↑ USDC contract. We only care about logs from THIS address.

       topics: [
         "0xddf252ad1be2c89b69c2b068fc378daa952ba7f163c4a11628f55a4df523b3ef",
          ↑ topic[0] is always keccak256(event signature).
            This specific hash always means "Transfer(address,address,uint256)".
            Hard-coded for all ERC-20 tokens forever.

         "0x000000000000000000000000OpenClawWalletAddress",
          ↑ topic[1] = 'from' address (padded to 32 bytes)
            The actual address is the LAST 40 hex chars (20 bytes).

         "0x000000000000000000000000PlatformWalletAddress",
          ↑ topic[2] = 'to' address (padded to 32 bytes)
            We check: does this match our platform wallet?
       ],

       data: "0x00000000000000000000000000000000000000000000000000000000000F4240"
              ↑ The transfer amount, ABI-encoded as uint256.
                0x000...0F4240 = 1,000,000 in decimal = $1.00 USDC.
                We decode this and check: is it ≥ what the tier costs?
     }
   ]

HOW PLATFORM READS IT (in plain English)
─────────────────────────────────────────

For each log in receipt.logs:
  1. Is log.address the USDC contract?          → skip if not
  2. Is log.topics[0] the Transfer topic hash?  → skip if not
  3. Is log.topics[2] our platform wallet?      → skip if not
  4. Decode log.data as uint256 → that's the amount
  5. Add to running total

After all logs:
  total ≥ tier.price_usdc_micro?  → payment confirmed ✓
  total = 0?                      → no transfer found, reject ✗
  total < tier price?             → underpayment, reject ✗
```

## Part 5: Replay Protection

Without protection, a malicious agent could:
1. Pay once with tx_hash `0xABC`
2. Call `/purchase/confirm` with `0xABC` → get a token
3. Call `/purchase/confirm` again with `0xABC` → get a second token for free

Protection is simple: **every tx_hash can only be used once.**

```
purchases table
───────────────
id           │ purchase_ref  │ tx_hash (UNIQUE)  │ status
─────────────┼───────────────┼───────────────────┼───────────
purch_1      │ pref_abc      │ 0xTxHash1         │ confirmed
purch_2      │ pref_def      │ 0xTxHash2         │ confirmed

If OpenClaw tries to submit 0xTxHash1 again:
  → DB unique constraint rejects the INSERT
  → Platform returns 400 { code: "tx_already_used" }
  → No second token issued
```

The same protection applies to `purchase_ref` — once confirmed, it cannot be used again.

## Part 6: The Refund (On-Chain)

If the platform fails to deliver a token after confirming payment, it sends USDC back.
This is the reverse of the purchase — the platform's wallet sends to the buyer's wallet.

```
                    ┌─────────────┐         ┌──────────────┐        ┌──────────────────┐
                    │  OPENCLAW   │         │   PLATFORM   │        │  BASE BLOCKCHAIN │
                    └──────┬──────┘         └──────┬───────┘        └────────┬─────────┘
                           │                       │                         │
  Purchase confirmed, payment received,            │                         │
  but JWT generation fails internally              │                         │
                           │                       │                         │
                           │                       │  Platform builds tx:    │
                           │                       │  ┌─────────────────┐    │
                           │                       │  │ USDC.transfer(  │    │
                           │                       │  │  0xOpenClaw,    │    │
                           │                       │  │  1_000_000)     │    │
                           │                       │  └─────────────────┘    │
                           │                       │                         │
                           │                       │──── broadcasts tx ─────►│
                           │                       │                         │
                           │                       │    USDC contract:       │
                           │                       │    Platform: -1 USDC    │
                           │                       │    OpenClaw: +1 USDC    │
                           │                       │                         │
                           │                       │◄────── tx_hash ─────────│
                           │                       │                         │
                           │                       │  DB: purchase.status    │
                           │                       │      = "refunded"       │
                           │                       │                         │
  OpenClaw's wallet shows  │                       │                         │
  +1 USDC automatically    │                       │                         │
  (blockchain state updated)                       │                         │
```

**The platform wallet needs ETH (not USDC) for gas when sending refunds.**
On Base, sending USDC costs ~$0.001 in ETH. Keep ~0.01 ETH in the platform wallet (covers 10,000 refunds).

## Part 8: Complete Picture

```
WHAT EXISTS WHERE
─────────────────

ON THE PLATFORM SERVER (our code):
  ├── Platform private key (env var, never leaves server)
  ├── PostgreSQL with purchases table
  ├── RSA private key for signing JWTs
  └── viem client connected to Base via Alchemy RPC

ON BASE BLOCKCHAIN (public, permanent):
  ├── USDC contract at 0x833589...
  │     └── Balances: { 0xPlatform: 2_200_000, 0xOpenClaw: 4_000_000, ... }
  └── Transaction history:
        └── 0x7f3a... (OpenClaw sent 1 USDC to Platform, block #12345678)

ON OPENCLAW AGENT:
  ├── Private key (agent's wallet — agent operator manages this)
  ├── Token Store { "openweather": { token: "eyJ...", remaining: 99 } }
  └── Platform catalog URL

WHAT THE PLATFORM ACTUALLY DOES:
  1. Tells OpenClaw where to send money (platform_wallet address in Agent Card)
  2. Checks that the money arrived (reads Base via viem)
  3. Issues a JWT proving access was paid for (RSA-signed, tamper-proof)
  4. Pays sellers their share (Stripe Connect transfer)

WHAT THE PLATFORM DOES NOT DO:
  × Touch OpenClaw's wallet
  × Hold OpenClaw's private key
  × Sit in the middle of API calls
  × Know OpenClaw's identity (only their wallet address)
```

## Part 9: Why Base and Not Ethereum?

```
┌──────────────┬──────────────┬───────────────┐
```
