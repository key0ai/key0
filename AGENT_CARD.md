# Agent Card Reference

This document explains all fields in the **Agent Card** format used by AgentGate. The Agent Card is a standardized JSON document published at `/.well-known/agent.json` that describes an agent's capabilities, pricing, and interaction methods according to the A2A (Agent-to-Agent) protocol.

## Overview

The Agent Card is automatically generated by `buildAgentCard()` from your `SellerConfig`. It serves as a machine-readable service discovery document that allows client agents to:

1. Discover available skills and their pricing
2. Understand payment requirements (USDC on Base)
3. Determine supported capabilities and extensions
4. Learn input/output formats

## Top-Level Fields

### `name`
- **Type**: `string`
- **Required**: Yes
- **Description**: Human-readable name of the agent
- **Example**: `"Photo API Agent"`
- **Set from**: `SellerConfig.agentName`

### `description`
- **Type**: `string`
- **Required**: Yes
- **Description**: Brief description of what the agent does or provides
- **Example**: `"Payment-gated access to premium photo collection"`
- **Set from**: `SellerConfig.agentDescription`

### `url`
- **Type**: `string`
- **Required**: Yes
- **Description**: Primary endpoint URL for accessing the agent's services
- **Example**: `"https://api.example.com/x402/access"`
- **Set from**: `${SellerConfig.agentUrl}/x402/access`

### `version`
- **Type**: `string`
- **Required**: Yes
- **Description**: Semantic version of the agent implementation
- **Example**: `"1.0.0"`
- **Default**: `"1.0.0"`
- **Set from**: `SellerConfig.version`

### `protocolVersion`
- **Type**: `string`
- **Required**: Yes
- **Description**: Version of the A2A protocol this agent implements
- **Example**: `"0.3.0"`
- **Set from**: Hardcoded to `"0.3.0"` in `buildAgentCard()`

---

## `capabilities`

Describes the agent's protocol capabilities and supported extensions.

### `capabilities.extensions`
- **Type**: `AgentExtension[]` (optional)
- **Description**: Array of protocol extensions this agent supports
- **Example**:
  ```json
  [
    {
      "uri": "https://a2a.org/extensions/x402",
      "description": "Supports x402 payments with USDC on Base mainnet.",
      "required": true
    }
  ]
  ```

#### `AgentExtension` fields:

##### `uri`
- **Type**: `string`
- **Required**: Yes
- **Description**: Unique identifier URI for the extension
- **Example**: `"https://a2a.org/extensions/x402"`

##### `description`
- **Type**: `string`
- **Required**: No
- **Description**: Human-readable explanation of the extension
- **Example**: `"Supports x402 payments with USDC on Base Sepolia."`

##### `required`
- **Type**: `boolean`
- **Required**: No
- **Description**: Whether this extension must be supported by clients
- **Example**: `true`

##### `params`
- **Type**: `Record<string, unknown>`
- **Required**: No
- **Description**: Extension-specific configuration parameters
- **Example**: `{ "maxRetries": 3 }`

### `capabilities.pushNotifications`
- **Type**: `boolean` (optional)
- **Description**: Whether the agent supports push notifications
- **Default**: `false`
- **Set from**: Hardcoded to `false` in AgentGate

### `capabilities.streaming`
- **Type**: `boolean` (optional)
- **Description**: Whether the agent supports streaming responses
- **Default**: `false`
- **Set from**: Hardcoded to `false` in AgentGate

### `capabilities.stateTransitionHistory`
- **Type**: `boolean` (optional)
- **Description**: Whether clients can query state transition history
- **Default**: `false`
- **Set from**: Hardcoded to `false` in AgentGate

---

## `defaultInputModes`
- **Type**: `string[]`
- **Required**: Yes
- **Description**: MIME types or content formats accepted as input
- **Example**: `["text"]` or `["application/json"]`
- **Set from**: Hardcoded to `["text"]` in AgentGate

---

## `defaultOutputModes`
- **Type**: `string[]`
- **Required**: Yes
- **Description**: MIME types or content formats returned as output
- **Example**: `["application/json"]`
- **Set from**: Hardcoded to `["application/json"]` in AgentGate

---

## `skills`

Array of `AgentSkill` objects describing available capabilities with pricing.

### `AgentSkill` fields:

#### `id`
- **Type**: `string`
- **Required**: Yes
- **Description**: Unique identifier for the skill
- **Example**: `"basic"`, `"premium"`, `"enterprise"`
- **Set from**: `ProductTier.tierId`

#### `name`
- **Type**: `string`
- **Required**: Yes
- **Description**: Human-readable name for the skill
- **Example**: `"Basic Access"`, `"Premium API"`
- **Set from**: `ProductTier.label`

#### `description`
- **Type**: `string`
- **Required**: Yes
- **Description**: Detailed explanation of what the skill provides
- **Example**: `"Pay $0.10 USDC for Basic access"`
- **Generated from**: `"Pay {amount} USDC for {label} access"`

#### `tags`
- **Type**: `string[]`
- **Required**: Yes
- **Description**: Categorization tags for the skill
- **Example**: `["payment", "x402"]`
- **Set from**: Hardcoded to `["payment", "x402"]` in AgentGate

#### `examples`
- **Type**: `string[]` (optional)
- **Description**: Example usage strings
- **Example**: `["Request access to photo-123 with basic tier"]`

#### `url`
- **Type**: `string` (optional)
- **Description**: Direct HTTP POST endpoint for this specific skill
- **Example**: `"https://api.example.com/a2a/access"`
- **Set from**: `${SellerConfig.agentUrl}${basePath}/access`

#### `inputSchema`
- **Type**: `AgentSkillInputSchema`
- **Required**: Yes
- **Description**: JSON Schema defining the expected input structure
- **Example**:
  ```json
  {
    "type": "object",
    "properties": {
      "type": {
        "type": "string",
        "description": "Must be 'AccessRequest'"
      },
      "tierId": {
        "type": "string",
        "description": "Product tier identifier"
      },
      "requestId": {
        "type": "string",
        "description": "Client-generated idempotency key (UUID)"
      }
    },
    "required": ["type", "tierId", "requestId"]
  }
  ```

##### `AgentSkillInputSchema` structure:
- `type`: Always `"object"`
- `properties`: Object mapping field names to their type definitions
  - Each property has:
    - `type`: JSON Schema type (`"string"`, `"number"`, `"boolean"`, etc.)
    - `description`: Human-readable explanation
- `required`: Array of required property names

#### `outputSchema`
- **Type**: `AgentSkillInputSchema`
- **Required**: Yes
- **Description**: JSON Schema defining the response structure
- **Example**:
  ```json
  {
    "type": "object",
    "properties": {
      "accessToken": {
        "type": "string",
        "description": "JWT token for API access"
      },
      "tokenType": {
        "type": "string",
        "description": "Token type (usually 'Bearer')"
      },
      "expiresAt": {
        "type": "string",
        "description": "ISO 8601 expiration timestamp"
      },
      "resourceEndpoint": {
        "type": "string",
        "description": "URL to access the protected resource"
      },
      "txHash": {
        "type": "string",
        "description": "On-chain transaction hash"
      },
      "explorerUrl": {
        "type": "string",
        "description": "Blockchain explorer URL"
      }
    }
  }
  ```

#### `pricing`
- **Type**: `SkillPricing[]` (optional)
- **Description**: Array of pricing information for this skill
- **Example**:
  ```json
  [
    {
      "tierId": "basic",
      "label": "Basic Access",
      "amount": "$0.10",
      "asset": "USDC",
      "chainId": 8453,
      "walletAddress": "0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb"
    }
  ]
  ```

##### `SkillPricing` fields:

###### `tierId`
- **Type**: `string`
- **Required**: Yes
- **Description**: Unique identifier for this pricing tier
- **Example**: `"basic"`, `"premium"`
- **Set from**: `ProductTier.tierId`

###### `label`
- **Type**: `string`
- **Required**: Yes
- **Description**: Human-readable name for this tier
- **Example**: `"Basic Access"`
- **Set from**: `ProductTier.label`

###### `amount`
- **Type**: `string`
- **Required**: Yes
- **Description**: Human-readable USD price (settled as USDC)
- **Format**: `"$X.XX"`
- **Example**: `"$0.10"`, `"$5.00"`
- **Set from**: `ProductTier.amount`

###### `asset`
- **Type**: `"USDC"`
- **Required**: Yes
- **Description**: Payment asset (currently only USDC is supported)
- **Set from**: Hardcoded to `"USDC"`

###### `chainId`
- **Type**: `number`
- **Required**: Yes
- **Description**: Blockchain chain ID
- **Values**:
  - `8453`: Base mainnet
  - `84532`: Base Sepolia (testnet)
- **Set from**: Derived from `SellerConfig.network` via `CHAIN_CONFIGS`

###### `walletAddress`
- **Type**: `string` (format: `0x${string}`)
- **Required**: Yes
- **Description**: Seller's wallet address that receives USDC payments
- **Example**: `"0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb"`
- **Set from**: `SellerConfig.walletAddress`

---

## `provider`

Information about the organization providing the agent.

### `provider.organization`
- **Type**: `string` (optional)
- **Description**: Name of the provider organization
- **Example**: `"Acme Corp"`
- **Set from**: `SellerConfig.providerName`

### `provider.url`
- **Type**: `string` (optional)
- **Description**: Website URL for the provider
- **Example**: `"https://acme.com"`
- **Set from**: `SellerConfig.providerUrl`

---

## `additionalInterfaces`

Optional array of alternative interfaces for accessing the agent.

### `AgentInterface` fields:

#### `url`
- **Type**: `string`
- **Required**: Yes
- **Description**: Endpoint URL for this interface
- **Example**: `"https://api.example.com/grpc"`

#### `transport`
- **Type**: `"JSONRPC" | "HTTP+JSON" | "GRPC"`
- **Required**: Yes
- **Description**: Protocol/transport type for this interface
- **Example**: `"HTTP+JSON"`

## How It's Generated

The Agent Card is automatically built by `buildAgentCard(config: SellerConfig)` in `src/core/agent-card.ts`. The function:

1. Maps `SellerConfig.products` to `skills` array (one skill per product tier)
2. Derives `chainId` from `SellerConfig.network` via `CHAIN_CONFIGS`
3. Constructs the x402 extension with network-specific description
4. Builds endpoint URLs from `agentUrl` and `basePath`
5. Returns a complete, valid Agent Card
